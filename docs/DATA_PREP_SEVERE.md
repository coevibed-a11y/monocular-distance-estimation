# 🌧 Severe 주행 환경 데이터 전처리 파이프라인 (Part 2)

## 1. create_val_severe.py

### 🔄 코드 전체 플로우 (데이터 흐름 기반)
1. **데이터 로드:** 악천후(눈/비/안개 등) 조건에서 정합이 완료된 도심 및 고속도로 환경 COCO JSON 데이터 세트 로드
2. **악천후 층화 분할:** 악천후 환경 특성상 극도로 편중된 객체 분포를 해결하기 위해 악천후 전용 층화 분할(Stratified Split) 알고리즘 구동
3. **최소 수량 보장:** 검증(Val) 및 평가(Test) 세트 내에 악천후 환경 희귀 클래스별 최소 객체 수(`MIN_OBJECTS = 30`)를 강제 보장하며 이미지 분배
4. **ID 오프셋 가산:** 기 구축된 맑은 날(Clear) 데이터셋과 ID가 절대 겹치지 않도록 고유 오프셋(Offset) 가산 후 독립적인 고유 ID 재발급
5. **독립 데이터 저장:** 최종 악천후 검증 및 평가용 `severe_val.json`, `severe_test.json` 데이터셋 분할 저장
6. **모듈 재사용 (핵심):** 실제 3D 기하학 투영 및 2D-3D IoU 정합 등의 세부 데이터 가공 프로세스는 앞서 Clear 파이프라인에 구축된 코드 플로우 및 전처리 모듈을 100% 동일하게 재사용하여 최종 데이터셋 생성 완료

### 💡 엔지니어링 의미
> **핵심 가치: 전처리 모듈의 추상화를 통한 코드 재사용성 극대화 및 악천후 강건성(Robustness) 검증 세트 확립**
> 초기 기하학 투영 및 IoU 정합 모듈을 환경 의존성 없이 유연하게 추상화하여 설계한 덕분에, 악천후 환경에서는 독립적인 진입점(`create_val_severe.py`)을 통해 데이터 밸런싱 분할과 ID 오프셋 제어만 수행한 뒤 후속 가공은 기존 코드를 그대로 재사용했습니다. 이를 통해 파이프라인 전체의 코드 중복을 최소화하고 유지보수성을 극대화했습니다. 또한 자율주행 비전 시스템에서 가장 치명적인 '악천후 환경에서의 인지 성능 강건성'을 독립적으로 평가할 수 있도록 신뢰성 높은 벤치마크 검증 세트를 확립했습니다.

### 🛠️ 핵심 코드 구조
* **`stratified_split_by_rare_classes()`**
  - 악천후 주행 조건 데이터 내에서 노이즈가 심한 희귀 카테고리를 우선 탐색하여 검증/평가 세트에 밸런싱 분배를 수행하는 함수입니다.
* **`apply_id_offsets_and_absolute_paths()`**
  - 악천후 데이터셋의 고유 ID 무결성을 확보하고 이미지 참조 실패를 방지하기 위해 절대 경로 변환을 수행하는 함수입니다.
