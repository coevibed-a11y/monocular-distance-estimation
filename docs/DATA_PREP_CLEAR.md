# ☀️ Clear 주행 환경 데이터 전처리 파이프라인 (Part 1)

## 1. pre_city_to_coco.py

### 🔄 코드 전체 플로우 (데이터 흐름 기반)
1. **원천 데이터 로드:** 외부 라벨링 툴(Superb AI 등)에서 수집된 비표준 도심 주행 2D JSON 파일 로드
2. **좌표 파싱:** 비표준 3중 중첩 리스트 구조의 `Segmentation` 좌표 순회 및 파싱
3. **포맷 표준화:** COCO 인스턴스 세그멘테이션 표준 포맷(`BBox`, `Area`, `Segmentation`)으로 재구조화
4. **확장 필드 주입:** 하이브리드 거리 추정 연산용 `distance: -1` 확장 필드 초기 주입
5. **최종 데이터 출력:** 단일 통합 COCO JSON 파일로 내보내기

### 💡 엔지니어링 의미
> **핵심 가치: 다운스트림 학습 파이프라인(Detectron2)의 런타임 에러 원천 차단**
> 외부 원천 데이터가 가진 비표준 가변성을 전처리 단계에서 정밀하게 파싱하여 리스크를 완전히 흡수합니다. 특히 데이터 타입 불일치 에러를 막기 위해 좌표 데이터를 `Float` 타입으로 업캐스팅하여 데이터 무결성을 확보하고, 거리 추정용 확장 인터페이스 설계를 선제적으로 완료했습니다.

### 🛠️ 핵심 코드 구조
* **`convert_raw_to_coco_format()`**
  - 비표준 3중 중첩 좌표계를 언패킹하여 COCO 표준 폴리곤 형태로 파싱하고 `distance` 확장 필드를 초기 바인딩하는 함수입니다.
* **`main()`**
  - 대상 경로의 모든 원천 JSON 파일을 스캔 및 순회하며 데이터를 취합하고 하나의 COCO 데이터셋 파일로 출력하는 메인 제어 루틴입니다.

## 2. pre_origin_to_coco.py

### 🔄 코드 전체 플로우 (데이터 흐름 기반)
1. **원천 데이터 로드:** 원본 고속도로 주행 데이터 JSON 파일 로드
2. **외곽선 추출:** 객체별 폴리곤 외곽선 좌표 추출
3. **정밀 면적 산출:** `pycocotools.mask` 모듈 연산을 통한 RLE 기반 정밀 픽셀 면적(`Area`) 산출 및 BBox 계산
4. **카테고리 정규화:** 수집 환경별로 상이한 클래스 명명법(예: `fence`, `twowheeler` 등)을 통합 카테고리 맵 사전에 맞춰 표준 ID로 일괄 치환
5. **최종 데이터 출력:** 최종 고속도로 환경 COCO JSON 저장

### 💡 엔지니어링 의미
> **핵심 가치: 픽셀 단위 정밀 Area 산출을 통한 COCO 포맷 정밀도 극대화**
> 단순히 가로×세로 연산으로 오차가 큰 사각형 면적을 구하는 방식 대신, RLE 인코딩을 거쳐 객체의 실제 형태를 픽셀 단위로 반영하는 정밀 면적(`maskUtils.area`)을 계산하여 레이블의 품질을 높였습니다. 또한 텍스트 정규화 사전을 설계하여 서로 분산 수집된 데이터셋 간의 의미적 클래스 무결성을 동시에 확보했습니다.

### 🛠️ 핵심 코드 구조
* **`polygon_to_bbox_and_area()`**
  - 입력된 폴리곤 좌표를 기반으로 사각형 BBox를 도출하고, RLE(Run-Length Encoding) 인코딩을 적용해 정밀 픽셀 면적을 계산하는 함수입니다.
* **`consolidate_categories()`**
  - 상이하게 기재된 텍스트 카테고리명을 소문자 정규화 및 사전 매핑을 통해 9개의 표준 Target Class ID로 통일하는 함수입니다.

## 3. pre_city_plus_dist1.py

### 🔄 코드 전체 플로우 (데이터 흐름 기반)
1. **데이터 로드:** 라이다 센서 기준의 3D 객체 라벨 데이터 및 카메라 캘리브레이션 파일(Intrinsic, Extrinsic 매트릭스) 로드
2. **동차좌표계 변환:** 행렬 곱 연산 구조를 맞추기 위해 3D 좌표 뒤에 1을 추가하여 변환
3. **외재 변환 (Extrinsic):** 외재행렬 연산을 통해 라이다 좌표계를 카메라 3D 좌표계로 변환
4. **내재 변환 (Intrinsic):** 내재행렬 연산을 통해 2D 초점 평면으로 투영
5. **원근 정규화 및 필터링:** 깊이($Z$축) 값을 분모로 나누어 최종 2D 픽셀 좌표 $(u, v)$를 산출하고 후방 왜곡 데이터 필터링 제거
6. **2D BBox 추출:** 투영된 점들을 감싸는 최소 사각형 2D BBox를 추출하여 중간 데이터로 저장

### 💡 엔지니어링 의미
> **핵심 가치: 이기종 센서(LiDAR-Camera) 간 기하학적 투영 정밀 정합 및 연산 최적화**
> 공간 차원이 다른 두 센서의 데이터를 캘리브레이션 매트릭스 연산 파이프라인을 통해 직접 기하학적으로 정합했습니다. 대용량 3D 공간 좌표 연산의 병목을 해결하기 위해 `multiprocessing.Pool` 기반 병렬 가속 연산을 도입했으며, 카메라 후방 영역 및 렌즈 왜곡 임계값 경계에 찍히는 기하 노이즈를 $Z \le 1.0\text{m}$ 스크리닝 조건을 통해 차단하여 데이터셋의 신뢰성을 확보했습니다.

### 🛠️ 핵심 코드 구조
* **`project_3d_to_2d()`**
  - 동차좌표계 변환, 외재/내재 행렬 곱, Z축 원근 정규화 연산을 통해 3D 포인트를 2D 이미지 픽셀 좌표로 변환하는 핵심 기하 연산 함수입니다.
* **`is_valid_bbox()`**
  - 투영 연산 결과 생성된 2D 박스가 이미지 해상도 범위를 극단적으로 벗어나거나 물리적으로 불가능한 크기를 가질 경우 노이즈로 판단하여 차단하는 방어 로직 함수입니다.

## 4. pre_city_plus_dist2.py

### 🔄 코드 전체 플로우 (데이터 흐름 기반)
1. **데이터 로드:** 3D 투영 박스 데이터와 1단계에서 빌드한 원본 2D COCO JSON 파일 로드
2. **박스 전수 순회:** 한 이미지 내의 원본 박스들과 투영 박스들을 전수 비교 순회
3. **클래스 1차 필터링:** 카테고리 매핑 테이블을 거쳐 클래스 명이 일치하는 객체 쌍만 필터링 수행
4. **IoU 계산:** 필터링을 통과한 두 박스 간의 IoU(Intersection over Union) 계산
5. **Best IoU 객체 매칭:** 임계값(`IOU_THRESHOLD = 0.5`)을 충족하는 객체 중 겹침 신뢰도가 가장 높은 객체를 동일 객체로 결정
6. **거리값 주입 및 마스킹:** 매칭 성공 시 실제 거리(`distance`) 값을 원본 COCO 데이터셋 내부에 동적 주입하고, 실패 시 `-1.0`을 주입하여 결측치 마스킹 처리

### 💡 엔지니어링 의미
> **핵심 가치: IoU 기반 데이터 정합(Data Association)을 통한 오차 보정 및 다운스트림 학습 최적화**
> 센서 간의 물리적 캘리브레이션 오차나 라벨링 공정 가공 오차로 인해 투영 박스와 원본 박스가 완벽히 일치하지 않는 문제를 2D-3D 정합 알고리즘으로 해결했습니다. 특히 정합 실패 데이터를 무조건 삭제하는 대신 `-1.0` 값으로 결측치 마스킹 바인딩 처리를 수행하여, 추후 학습 모델의 커스텀 로스 함수에서 유효 마스크 조건(`valid_mask = distance > -0.5`)을 통해 연산 가중치를 동적으로 제어할 수 있도록 아키텍처를 선제 설계했습니다.

### 🛠️ 핵심 코드 구조
* **`calculate_bbox_iou()`**
  - 두 Bounding Box의 좌표 정보 체계를 비교하여 교집합 면적을 합집합 면적으로 나누어 겹침 신뢰도를 연산하는 함수입니다.
* **`match_and_inject_distance()`**
  - 원본 2D 카테고리와 투영 카테고리의 일치성을 검증하고, 최적 매칭 결과를 COCO 포맷의 딕셔너리 구조에 `distance` 값으로 할당 및 동기화하는 메인 정합 함수입니다.

## 5. create_val_clear.py

### 🔄 코드 전체 플로우 (데이터 흐름 기반)
1. **데이터 로드:** 정합이 완료된 도심 및 고속도로 COCO JSON 데이터 세트 로드
2. **층화 분할 구동:** 전체 데이터에서 빈도가 낮은 희귀 클래스(보행자, 자전거 등)를 역순으로 추적하는 층화 분할(Stratified Split) 알고리즘 구동
3. **최소 수량 보장:** 검증(Val) 및 평가(Test) 세트 내에 각 클래스별 최소 객체 수(`MIN_OBJECTS = 30`)를 강제 할당 및 이미지 분배
4. **고유 ID 재발급:** 이기종 데이터 병합 시 인덱스가 겹치지 않도록 `image_id`와 `annotation_id`에 동적 오프셋(Offset) 가산 후 고유 ID 재발급
5. **경로 동적 변환:** 로더 가속화를 위해 데이터 내 파일 경로를 런타임 절대 경로로 동적 변환
6. **최종 데이터 분할 저장:** 최종 `train.json`, `val.json`, `test.json`으로 분할 저장

### 💡 엔지니어링 의미
> **핵심 가치: 층화 분할을 통한 클래스 불균형 해소 및 데이터 병합 무결성 확보**
> 자율주행 도메인의 고질적인 문제인 클래스 불균형(차량 편중, 보행자 부족)으로 인해 검증 세트에서 특정 클래스가 누락되어 평가 지표의 신뢰성이 무너지는 현상을 직접 구현한 층화 분할 알고리즘으로 방어했습니다. 또한 상이한 원천 데이터를 통합할 때 발생하는 데이터베이스 기본 키(Primary Key) 충돌을 오프셋 메커니즘으로 제어하여 데이터 파이프라인의 무결성을 완전하게 확보했습니다.

### 🛠️ 핵심 코드 구조
* **`stratified_split_by_rare_classes()`**
  - 희귀 카테고리를 우선 순회하며 지정된 최소 객체 수량을 충족할 때까지 검증/평가 세트에 데이터를 강제 분배하고 잔여 데이터를 훈련 세트로 넘기는 균형 분할 함수입니다.
* **`apply_id_offsets_and_absolute_paths()`**
  - 병합 대상 데이터 세트 간 고유 ID가 충돌하지 않도록 고유 오프셋을 더하고, 로컬 디렉토리 환경에 맞춰 이미지 경로를 절대 경로로 일괄 갱신하는 함수입니다.

## 6. test.py

### 🔄 코드 전체 플로우 (데이터 흐름 기반)
1. **데이터 로드:** 최종 생성된 훈련/검증용 통합 COCO JSON 파일 로드
2. **어노테이션 순회:** `annotations` 리스트 전수 순회
3. **필드 스캔:** 각 객체 노드 내에 존재하는 `distance` 필드 스캔
4. **통계 계산:** 유효 거리 값($\text{distance} > 0$)을 가진 객체의 총수와 비율 계산
5. **결과 출력:** 무결성 검증 통계 로그 출력

### 💡 엔지니어링 의미
> **핵심 가치: 사전 자가 진단(Sanity Check)을 통한 파이프라인 무결성 검증 및 자원 낭비 방지**
> 수많은 투영 및 정합 연산을 거친 최종 대용량 데이터 세트가 모델 학습에 바로 투입될 수 있는 상태인지 검증하는 자가 진단 파이프라인입니다. 막대한 컴퓨팅 자원이 소모되는 딥러닝 학습 루프에 진입하기 전, 전처리 데이터셋 내부의 유효 거리 데이터 비율을 초고속으로 검증하여 가중치 에러 및 파이프라인 휴먼 에러를 미연에 방지합니다.

### 🛠️ 핵심 코드 구조
* **`verify_dataset_integrity()`**
  - 생성된 JSON 레이블 데이터를 파싱하여 유효 정합 카운트를 집계하고, 전체 데이터 대비 성공률 통계를 계산하여 데이터 무결성을 시각화하는 모니터링 함수입니다.
